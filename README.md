# Galactic Trading Post

Galactic Trading Post is a mobile-first browser idle/management game built with plain HTML, CSS, and JavaScript.

You run a trading post in a home system, buy cargo, send merchant ships on round-trip trade missions, scout new systems, manage ship upgrades, and react to temporary market events. Game state is saved in `localStorage`, and the app is configured to work as an offline-capable PWA once it has been loaded from a real web origin.

## Features

- Single-page mobile-first UI
- Local save/load with `localStorage`
- Offline progression for missions, scouting, upgrades, and market decay
- Installable/offline-ready PWA setup
- Dynamic supply-and-demand pricing
- Timed system-specific market events with a live newsfeed
- Round-trip trade planning with outbound and return cargo
- Per-ship customization:
  - custom ship names
  - custom name colors
  - cargo upgrades
  - speed upgrades
- Explicit ship selection for trade missions

## How the Economy Works

- The player buys cargo in the home system.
- A merchant ship carries that cargo to a discovered destination system.
- The ship then buys a return cargo in the destination system and comes back home.
- Credits are affected by:
  - outbound cargo purchase
  - return cargo escrow reservation
  - sale of outbound cargo at destination
  - sale of return cargo at home
- Prices are influenced by:
  - system economic profile
  - temporary supply/demand shifts from buying and selling
  - temporary market events from the newsfeed

## Star Systems and System Types

- There are 6 commodities:
  - Food
  - Water
  - Ore
  - Fuel
  - Electronics
  - Medicine
- System types are generated by:
  - choosing 2 high-priced commodities from the 6
  - choosing 2 low-priced commodities from the remaining 4
- This creates all valid non-overlapping system types.
- Each system has:
  - a name
  - a type
  - a distance in LY
  - a discovered flag
  - dynamic prices

## Merchant Ships

Each merchant ship is unique and can be managed individually in the Hangar.

- Ships can be renamed.
- Ship name colors can be customized.
- Ships can be upgraded:
  - Cargo upgrades increase carrying capacity.
  - Speed upgrades reduce mission duration.
- Upgrade costs scale per ship and per upgrade level.
- Ships are unavailable for trade while upgrading.

## Market Events

The game runs a lightweight event/news system:

- Two events are active at a time.
- Events last 20 minutes.
- New events start every 10 minutes.
- Each event affects one commodity in one discovered system.
- Events raise or lower that commodity by one market tier.
- Market tiers include:
  - Very Low
  - Low
  - Normal
  - High
  - Very High

## Running the Game

You can open `index.html` directly in a browser for basic local play.

For full PWA behavior, including service worker caching and installability, serve the project from a local web server instead of opening it with `file://`.

Examples:

```bash
python3 -m http.server 8000
```

Then open:

```text
http://localhost:8000
```

## Offline PWA Notes

- The service worker caches the app shell after the first successful load.
- After that, the game can continue to open offline.
- Install prompts and service worker support require `http://` or `https://`.
- `file://` does not support normal PWA behavior.

## Save Data

- Save data is stored in the browser with `localStorage`.
- Resetting the game from the UI clears the current save.
- Save key:
  - `galacticTradingPostSave_v1`
